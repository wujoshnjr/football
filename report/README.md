# Reports

This folder stores generated reports for validation, model monitoring, and data quality checks.

## Expected files

```text
report/
  data_sources_report.json
  model_card.md
  validation_summary.json
  backtest_summary.json
```

## Rules

- Reports should be generated by scripts when possible.
- Every report should include a timestamp and model or source version.
- A source failure should be recorded in the report instead of crashing the pipeline.
- Generated reports help track deployment readiness over time.
